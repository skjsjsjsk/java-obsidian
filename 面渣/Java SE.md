- Java数据类型以及对应的包装类和它们各自的大小: 
	- 分为基本数据类型(八种)和引用数据类型
		- 基本数据类型: 布尔型(boolean(1或4字节)), 数值型(整型(short(2字节), int(4字节), long(8字节)), 浮点型(float(4字节), double(8字节)), 字节型(byte(1字节))), 字符型(char(2字节))
			- ![300](assets/Java%20SE/file-20260317185339972.png)
		- 引用数据类型: 数组, 类, 接口
	- 包装类: Boolean, Character, Byte, Short, Integer, Long, Float, Double
		- 包装类是对象, 存在于堆中, 而基本数据类型是值, 存在于栈中
	- Integer默认的缓存大小为-128 - 127, 当用Integer创建对象超出该范围, 那么就会创建新对象
- 自动拆箱和装箱: 将基本数据类型转换为包装类就是装箱, 反过来就是拆箱
- 面向对象编程的特性: 封装, 继承和多态
	- 封装: 就是把数据和操作数据的方法捆绑到一起, 形成一个新的对象, 也就是类
	- 继承: 一个类(子类)可以继承父类的属性和方法, 同时它也可以重写和扩展父类的方法, 这就形成了多态
	- 多态: 不同的对象对用一种消息做出不同的行为
		- 实现多态的条件: 继承父类, 子类重写父类方法, 父类引用指向子类对象
- 重载和重写: 
	- 重写: 子类继承父类的方法, 其他所有的都相同, 但是方法体不同
	- 重载: 一个类有多个名字相同但参数列表不同
	- 设计原则: 
		- 重写还运用了**李氏代换原则**, 就是所有父类出现的地方, 子类一定可以出现
		- 除此之外, 还有**开闭原则**: 一个类要对扩展开放, 对修改关闭, 就是不要在源代码上进行修改, 一个类应该通过扩展来增加功能
		- **单一职责原则**: 一个类应该只有一个能引起它变化的原因, 这意味着一个类只负责一项职责, 脏页使得类更清晰, 更易维护
		- **接口隔离原则**: 客户端不应该依赖它所不需要的接口, 这说明设计接口时要尽量精简
		- **依赖倒置原则**: 高层模块不应该依赖于底层模块, 二者都应该依赖于其抽象; 抽象不应该依赖于细节, 细节应该依赖于抽象. 这意味着设计时应该尽量依赖抽象类和接口, 而不是实现类
- 抽象类和接口: 一个类只能继承一个抽象类, 但可以实现多个接口; 抽象类是is-a的关系, 耦合性强, 而接口是has-a的关系,比如一个类想要实现序列化, 那么只需要实现Serializable接口, 而不是继承一个序列化类; 抽象类更多是为多个相关的类提供一个基本的框架, 实现了初始化, 而接口是定义了一套行为规范
	- 类不能多继承是因为, 类如果多继承, 那么在调用与父方法相同的方法时, 不知道该调用谁的
	- 接口可以多继承, 是因为, 接口只是个行为规范, 它里面没有具体的实现
- 为什么重写了equals必须重写hashCode方法: 这是为了保证对象逻辑上的一致性, 比如HashMap, 因为它是根据对象的哈希码来被存储在桶中的, 它在查找时会根据key的哈希码确定在哪个桶里, 然后通过equals方法来找到.  如果重写了equals方法但是hashCode方法没有办法, 相同的对象有不同的哈希码, 从而导致无法正确处理
	- 如果equals相等但是hashCode不同, 那么可能会导致put的时候, 比如new出来的p1, p2两个逻辑上一样的人会被存储在哈希表的不同地方, 这导致了重复元素或键值冲突等等问题
- String, StringBuffer, StringBuilder: String是不可变的, 它每次拼接会生成一个新对象, 在大量字符串操作的情况下会造成极大的性能开销; 而builder是在原来对象本身上进行修改操作的, 但是它是线程不安全的, 适合单线程, 而buffer是线程安全的, 在方法上加了synchronized关键字, 其余与builder一样
	- `String s = "a"` vs `String s = new String("a")`的区别:
		- 第一个, Java会直接在字符串常量池中去查找是否有"a", 如果有, 则把池里的引用赋给"a", 没有就创建一个新的然后赋给"a"
		- 第二个, 会先去池中查找是否有"a", 如果有就引用它, 没有则创建一个, 第二步, 去堆中再创建一个新的字符串对象, 将它初始化为一个副本
