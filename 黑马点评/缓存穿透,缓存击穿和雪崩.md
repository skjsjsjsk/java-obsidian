# 缓存穿透
- 缓存穿透是指客户端请求的数据在缓存中和数据库中都不存在(可能会有故意查找不存的那个,导致数据库一直被访问,然后宕机)，这样缓存永远不会生效，这些请求都会打到数据库,压垮数据库.

- **接口限流与校验**：对恶意高频请求限流，校验请求参数合法性。
# 缓存击穿
- 缓存击穿问题也叫热点Key问题，就是一个被高并发访问并且缓存重建业务较复杂的key突然失效了，无数的请求访问会在瞬间给数据库带来巨大的冲击。
  1.([[实现redis分布式锁]])
   2 . **逻辑过期**：缓存存储时，为 value 添加 “逻辑过期时间”（不设置 Redis 的真实过期时间），查询时判断逻辑过期：若未过期：直接返回缓存数据；若已过期：先返回旧数据，同时异步启动线程重建缓存（重建时加分布式锁，避免重复重建）。
# 雪崩
- 指大量缓存 key 同时过期或 Redis 集群故障，导致请求集中冲击数据库。
  1.**过期时间随机化**：为不同 key 设置随机过期时间，避免同时失效。
  2.**缓存集群高可用**：部署 Redis 主从 + 哨兵或集群模式，避免单点故障。
  3.**熔断降级**：使用 Hystrix 组件，当缓存失效时限制数据库请求流量，返回默认值或错误提示。
  4.**预热缓存**：提前加载热点数据到缓存，避免冷启动时的缓存缺失。
