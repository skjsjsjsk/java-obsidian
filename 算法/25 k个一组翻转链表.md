- 思路就是创建一个翻转链表的方法 `reverse`, 然后定义一个用来寻找第K个节点的临时变量`tail`, 然后具体实现看下面注释
```
class Solution {
    public ListNode reverseKGroup(ListNode head, int k) {
        // 初始化一个节点tail, 这个节点用来寻找链表中的第k个节点, 判断是进行翻转链表还是返回原链表
        ListNode tail = head;
        for(int i=1; i<=k; i++){
            if(tail==null) return head;
            tail = tail.next;  // 这里的tail在遍历完出来后是第k个节点的下一个节点
        }
        
        // 下一步就是翻转这k个节点, 同时返回了新链表的头节点
        ListNode newHead = reverse(head, tail);
        
        // 因为经过第一次的前k个节点的翻转以后, reverse方法中的head与tail是同一个位置(第k个节点的下一个节点), 但是这个head只是reverse方法里面一个变量, 在外面的全局变量上, head仍然是原链表的头节点1, 又因为在代码一开始用tail这个局部变量存储了head, 所以在下面的递归里面存入tail这个局部变量即第k个节点的下一个节点, 而不是用head
        // 举个例子, 有5个节点的链表, 在第一次运行到这里时, 最外面的newNode是2->1->不确定, 这里的head=1, tail=3 
        // 进入了递归, 递到第二次到这里时, 这个递里面的newNode是4->3->不知道, 这里的head=上面传入的tail=3, 新的tail=5, 
	    // 进行第三次递, 发现不够2个, 所以返回head, 这里head=5, 是上面第二次递传入的tail(5), 
	    // 然后开始归, 第一次归, 有了newNode是4->3->5, 
	    // 第二次归就把这个街道了最外面的newNode的后面, 也就是2->1->4->3->5
	    
        head.next  = reverseKGroup(tail, k);
        return newHead;
    }
    private static ListNode reverse(ListNode head, ListNode tail){
        ListNode pre = null;  // 用的翻转链表里面的迭代法(头插法)
        ListNode nxt;
        // 因为tail是第k个节点的下一个节点, 所以使用不等于就可以成功让这k个节点全部翻转
        while(head != tail){
	        nxt = head.next;
            head.next = pre;
            //将这两位全部往前面挪一位 
            pre = head;
            head = nxt;
        }
        // 此时的pre就是新链表的头节点
        return pre;
    }
}
```
