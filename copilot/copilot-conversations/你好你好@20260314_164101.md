**ai**: 这是一个非常经典的面试题，也是理解 Spring 核心的基础。你可以这样回答：

**1. IOC (Inversion of Control) - 控制反转**

*   **定义**：IOC 是一种设计思想。它指的是将对象的创建、管理和依赖关系的处理权利，从程序员手中转移到了 Spring 容器手中。
*   **理解**：
    *   **传统方式**：如果在代码里需要用某个对象，我们通常要手动 `new` 一个，比如 `UserService service = new UserService();`。这时候，对象创建的**控制权**在程序员自己手里。
    *   **IOC 方式**：我们不再自己手动创建对象，而是告诉 Spring 容器“我需要这个对象”，Spring 容器帮你创建好并管理起来（比如通过 `@Component`、`@Service` 等注解标记的对象）。
*   **目的**：主要是为了**解耦**。对象之间不再直接依赖具体实现，而是依赖 Spring 容器提供的抽象（Bean），降低了代码的耦合度。

**2. DI (Dependency Injection) - 依赖注入**

*   **定义**：DI 是 IOC 思想的具体**实现方式**。
*   **理解**：意思是 Spring 容器在创建对象（Bean）的时候，会自动将这个对象所依赖的其他对象（属性）“注入”进去。
*   **举例**：比如 `UserController` 依赖 `UserService`。在传统方式下，你需要在 `UserController` 里写代码 `this.service = new UserService()`。而在 DI 中，我们只需要在 `UserController` 中通过 `@Autowired` 标记 `UserService`，Spring 容器在创建 `UserController` 时，就会自动把创建好的 `UserService` 对象“塞”给它。
*   **常见注入方式**：构造器注入（推荐）、Setter 注入、字段注入。

**3. 总结两者的关系**

可以用一句话总结：**IOC 是目标（把控制权交出去），DI 是手段（通过注入的方式来实现这种控制权的转移）。**

（**面试加分项**：你可以打个比方——IOC 就像是“找对象”，以前你需要自己到处去认识（手动 new），现在把你的需求告诉婚介所（Spring 容器），婚介所直接给你安排（DI）合适的人选。）

### 1. **Bean 的生命周期**（超级高频，必问）
这是 Spring 中最经典的八股文之一。
- **考察点**：看你知不知道 Spring 是如何从创建一个对象到销毁它的全过程。
- **回答思路**：主要说“实例化 -> 属性赋值 -> 初始化 -> 销毁”这四个大阶段，中间穿插 `BeanPostProcessor`（后置处理器）的作用。

### 2. **Spring 的事务**（高频）
业务开发离不开事务，所以问得很多。
- **常见问题**：
    - `@Transactional` 注解失效的场景有哪些？（这个很重要）
    - Spring 事务的传播机制（Propagation）有哪些？`REQUIRED` 和 `REQUIRES_NEW` 有什么区别？
    - Spring 事务是如何工作的？（基于 AOP 实现）

### 3. **Spring 循环依赖**（高频，中高级必问）
- **常见问题**：什么是循环依赖？Spring 是如何解决循环依赖的？三级缓存的作用是什么？
- **考察点**：看你有没有深入理解 Spring 源码或内部机制。这是区分初级和中级工程师的关键点。

### 4. **Bean 的作用域**
- **常见问题**：Spring Bean 默认的作用域是什么？单例和原型有什么区别？
- **考察点**：对应你笔记里提到的 `@Scope`。

### 5. **常用注解的区别**
- **常见问题**：`@Autowired` 和 `@Resource` 的区别？
- **回答思路**：`@Autowired` 是 Spring 的，默认按类型装配；`@Resource` 是 JDK 的，默认按名称装配。

### 6. **Spring MVC 的执行流程**
如果简历上写了 Web 项目，这个很可能会问。
- **常见问题**：一个请求从浏览器发过来，到 Controller 返回数据，中间经过了哪些组件？
- **关键点**：DispatcherServlet（核心控制器）、HandlerMapping（找控制器）、HandlerAdapter（执行控制器）、ViewResolver（视图解析器）。

### 7. **设计模式**
就像你笔记里记的，Spring 用了大量设计模式。
- **常见问题**：Spring 中用到了哪些设计模式？
- **回答思路**：工厂模式、单例模式、代理模式、模板方法模式、观察者模式等。

### 8. **Spring Boot 自动装配**（如果简历写了 Boot）
- **常见问题**：Spring Boot 的自动装配原理是什么？`@EnableAutoConfiguration` 做了什么事？
