## 动态规划
- 该题是一道经典的动态规划题, 为此我们需要**定义状态**, 写出**状态转移方程**, 和**赋值初始值**
- 关于定义状态, 我们要保证定义好的状态是**无后效性**的, 使得子问题的描述不会有不确定的地方
	- 像该题, 我们假设所给数组是`[-1, 2, 1, 10, 4]`, 如果我们设定子问题为, 经过`nums[i]`的连续子数组的最大和, 那么子问题就是: 
		- 子问题1: 经过-1的连续子数组的最大和
		- 子问题2: 经过2的连续子数组的最大和
		- 等等........
	- 虽然子问题设置出来了, 但子问题间的联系不太好看到, 也就是子问题的描述有不确定的地方, 这就是有后效性

	- 所以要重新定义子问题, 现在问题是不知道`nums[i]`是当前连续数组中的第几个元素, 比如:
		- 「经过 −3 的连续子数组」我们任意举出几个：
			- `[-2,1,-3,4]` ，−3 是这个连续子数组的第 3 个元素；
			- `[1,-3,4,-1]` ，−3 是这个连续子数组的第 2 个元素；
			- ……
	- 所以重新定义子问题为-3为该连续子数组的最后一个元素时的最大和, 即为**以`nums[i]`结尾的连续子数组的最大和**
	- 设`dp[i]`为**以`nums[i]`结尾的连续子数组的最大和**
	
	- 此时子问题定义完了, 寻找子问题间的规律, 发现以`dp[i]`就等于以`dp[i-1]`加上`nums[i]`, 而且如果`dp[i-1]<0`, 那么`dp[i] = nums[i]`, 要不然加上就会变小
		- 此时就发现了状态转移方程: 
			- ![](assets/7%2053最大子数组和/file-20260327151533308.png)

	- 接下来就该赋值初始值了, `dp[0]` 根据定义，只有 1 个数，一定以 `nums[0]` 结尾，因此 `dp[0] = nums[0]`
```
class Solution {
    public int maxSubArray(int[] nums) {
        int len = nums.length;

        // dp[i]就是以nums[i]结尾的连续子数组的最大和 
        int[] dp = new int[len];
        dp[0] = nums[0];
        
        // res是用来求最大值的
        int res = dp[0];

        // 判断设定的状态转移方程的情况, 得出结果
        for(int i = 1; i < len; i++){
            if(dp[i - 1] > 0){
                dp[i] = dp[i - 1] + nums[i];
                res = Math.max(res, dp[i]);
            }else{
                dp[i] = nums[i];
                res = Math.max(res, dp[i]);
            }
        }
        return res;
    }
}
```

## 前缀和加贪心
```
class Solution {
    public int maxSubArray(int[] nums) {
        int minPreSum = 0;
        int preSum = 0;
        int result = Integer.MIN_VALUE;
        for(int num: nums){
        
            // 首先先计算当前的前缀和
            preSum += num;
            
            // 接下来这部分就是贪心
            // 更新结果, 求出连续子数组(preSum - minpreSum)与result的最大值
            result = Math.max(result, preSum - minPreSum);
            
            // 更新最小前缀和
            minPreSum = Math.min(preSum, minPreSum);
        }
        return result;
    }
}
```
